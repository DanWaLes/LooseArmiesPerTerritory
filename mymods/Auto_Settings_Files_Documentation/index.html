The goal of this is to automatically generate the Client_PresentConfigureUI, Client_SaveConfigureUI and Client_PresentSettingsUI lua files. All these files should be left as-is. A settings.lua file will need to be created/edited.

settings.lua file syntax

the file must have a getSettings function. returns settings. settings is an array of settings and setting templates

Setting

  • must have a name - string - all names must be unique to function correctly
  • must have a inputType - one of 'int', 'float' or 'bool'
  • must have a defaultValue - any type - the default value for the setting's value in Client_PresentConfigureUI
  • must have a label - string - summery of what the setting does
  • can have a bkwrds value - for if the mod is already public and turned something that wasnt configurable before into a setting
  • can have a help(parent) function. is a more detailed explanation of what the setting does. appears as a ? next to the setting name. help is initially collapsed. every second click is collapsed again (after being expanded)

  • if inputType is 'int' or 'float' then

    • setting must have minValue - lowest allowed value while saving settings
    • setting must have maxValue - maximum allowed value while saving settings
    • setting can have absoluteMax - if used the settings slider wont go higher than maxValue, but while saving settings the setting value must be below this

  • if inputType is 'float' then

    • must have dp - int - the number of decimal places to round the setting value to when saved

  • if inputType is 'bool' then

    • can have subsettings. subsettings defined same way as settings

Setting templates

In the event of wanting to have infinite groups of settings, setting templates can be used. any settings that require this should have subsettings
  • must have a name - string - all names must be unique to function correctly
  • must have a isTemplate flag - always true
  • must have btnText - string - what the appears on the button to add a new group
  • must have a get(n) function. must return setting. n starts at 1
  • can a bkwrds value - for if the mod is public and goes from limited to unlimited amounts of setting groups. bkwrds number of setting groups will be generated. to avoid nil in Client_PresentSettingsUI

Accessing setting values

Each setting is stored in Mod.Settings[name]
