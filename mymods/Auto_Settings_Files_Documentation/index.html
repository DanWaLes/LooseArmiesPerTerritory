The goal of this is to automatically generates the Client_PresentConfigureUI, Client_SaveConfigureUI and Client_PresentSettingsUI lua files. All these files should be left as-is. A settings.lua file will need to be created.

settings.lua file syntax

the file must have a getSettings function. returns settings

Settings

settings is an array of setting
  • setting must have a name - string - all setting names must be unique to function correctly
  • setting must have a inputType - one of 'int', 'float' or 'bool'
  • setting must have a defaultValue - any type - the default value for the setting's value in Client_PresentConfigureUI
  • setting must have a label - string - summery of what the setting does
  • setting can have a bkwrds value - for if the mod is already public and turned something that wasnt configurable before into a setting
  • setting can have a help(parent) function. is a more detailed explanation of what the setting does. appears as a ? next to the setting name. help is initially collapsed. every second click is collapsed again (after being expanded)
  • setting can have a isTemplate flag. should only be true. for templates only

  • if inputType is 'int' or 'float' then

    • setting must have minValue - lowest allowed value while saving settings
    • setting must have maxValue - maximum allowed value while saving settings
    • setting can have absoluteMax - if used the settings slider wont go higher than maxValue, but while saving settings the setting value must be below this

  • if inputType is 'float' then

    • setting must have dp - int - the number of decimal places to round the setting value to when saved

  • if inputType is 'bool' then

    • setting can have subsettings. subsettings defined same way as settings

Templates

In the event of wanting to have infinite groups of settings, setting templates can be used. templates is a table indexed by names.
  • templated settings must have btnText - string - what the appears on the button to add a new group
  • templated settings must have a get(id) function. must return setting
  • templated settings must have a name (same as name key) and isTemplate (always true) - would recommend looping over templates to add them
  • templated settings can a bkwrds value - for if the mod is public and goes from limited to unlimited amounts of setting groups. bkwrds number of setting groups will be generated. to avoid nil in Client_PresentSettingsUI

Accessing setting values

Each setting is stored in Mod.Settings[name]
